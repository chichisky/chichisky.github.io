关于tabbar的一个需求

发表于 2021-06-04 | 分类于 需求问题

2021年6月4号的面试，去了直接配了一个vue项目，完成需求，最后一个需求，tabbar每隔一分钟跳转一次路由,跳转到当前路由的时候，打印出当前路由的名称及上个路由所剩的时间

总共可细分为3个小需求

  • 打印当前路由
  • 每隔一分钟跳转路由
  • 打印上个路由所剩的时间

打印当前路由

// 利用全局守卫可以完成
router.beforeEach((to, from, next) => {
  /*to and from are Route Object,next() must be called to resolve the hook */
  if(to.name != from.name){
    console.log(to.name);
    next()
  }
})

每隔一分钟跳转一次路由

  • 这里写的比较简陋，想法不知道对不对，但是基本实现了，在创建组件的时候执行一个函数，该函数60秒后自动跳转到下个路由
<script>
export default {
  methods: {
    Jump() {
      setTimeout(()=> {
        this.$router.push('/category')
      },60000)
    }
  },
  created () {
    this.Jump();
  }
}
</script>

打印上个路由剩下的时间

store.js

export default new Vuex.Store({
  // 定义了两个全局变量，一个是时间戳，另一个是当前路由，初始值为'/home'
  state: {
    currentTime: undefined,
    addRoute: '/home',
  },
  // 定义了两个方法，一个用来获取时间戳，改变state.currentTime
  // 另一个用来获取当前的路由
  mutations: {
    default_time (state) {
      state.currentTime = Date.parse( new Date ())/1000;
    },
    updateRoute(state, payload) {
      state.addRoute = payload;
    }
  }
})

router.js

router.beforeEach((to, from, next) => {
  /*to and from are Route Object,next() must be called to resolve the hook */
  // 判断的当前路由是否等于要去的路由，如果不想等，则获得当前的时间戳，打印剩余时间
  if(store.state.addRoute != to.path) {
    const currentTime = Date.parse(new Date())/1000;
    console.log(60 - currentTime + store.state.currentTime);
  }
  // 更新当前路由
  store.commit('updateRoute', to.path)
  // 打印当前路由名称，及重新赋值全局的时间戳。
  if(to.name !== from.name){
    console.log(to.name);
    store.commit('default_time');
    next()
  }
})
