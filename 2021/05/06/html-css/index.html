day01HTML简单介绍

一、网页的基本元素

  • 一个完整的HTML结构包括哪几部分？

    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charest="UTF-8">
    <title>Title</title>
  </head>
  <body>
  </body>
</html>

  • 文档声明

    • HTML文档声明，告诉浏览器当前页面是HTML5页面，让浏览器用HTML5的标准去解析识别HTML文档
    • 必须放到HTML文档的最前面，不能省略，省略了会出现兼容性问题

  • html元素

    • html元素是HTML文档的根元素，一个文件只能有一个，其他所有元素都是它的后代元素
    • lang => language 语言的意思，作用：
      • 帮助翻译工具确定要使用的翻译规则
      • 帮助语音合成工具确定要使用的发音
    • lang=”en” =>英文; lang=”zh” =>中文
      • zh-CN:地区限制匹配规范，表示用在中国大陆区域的中文
      • zh-Hans:语言限制匹配规范，表示简体中文

  • head元素

    • head元素里面的内容是一些”元数据”(元数据：描述数据的数据)
      • 元数据 => metadata;
        用来描述数据的数据
        对网页进行一些基本设置
    • 比如字符编码、网页标题、网页图标
      • meta charest=”UTF-8” => 字符集
        作用:将文字存储到计算机中，之后解析出来显示
        应用:所有的网页目前都需要采用UTF-8编码，所有浏览器在解析时，我们也要告诉浏览器当前我们使用的时UTF-8，浏览器才能正常的解析出来文字
        中文 => GB2312 => GBK
        其他国家：其他文字 => 自己的编码规则
        Unicode => 规则
        utf-16 => utf-32 => utf-8
        哈哈 => utf8 => 01010101 =>GBK 乱码
      • title => 网页标题
      • link:favicon 网页图标

  • body元素

    • 网页的具体内容和结构

二、基本标签

  • h元素：定义了六个大小不同的标题，加粗文本

  • strong：加粗

  • img：显示图片

    • src：图片的路径(网络图片/本地图片(相对路径/绝对路径))
    • alt：当图片无法加载，显示的代替文字
    • 注意：img如果只设置了width或height，浏览器会自动根据图片宽高比计算出height或width

  • a元素：定义锚点

    • href：超链接
      • target：
        _self(默认值):自己
        _blank:空白，新的标签页
      • 在iframe标签内的target：
        _parent:父元素
        _top:多层嵌套iframe，在顶层
        某个frame的name值：在某个frame中打开URL
      • “#id”:可以定义到id所在位置
      • 伪链接：没有指明具体链接地址的链接
        点击链接后具体做什么,可以编写对应的JS代码
        “#” onclick = “return false”
        “javascript: alert()” //弹出弹窗

  • iframe: 框架，一般用来包含别的页面

  • pre元素：预设文本，按照代码块中的文本格式渲染到浏览器上

3.2. SEO优化

SEO: 搜索引擎优化

  • 百度/搜狗/360/Google
  • h元素有助于网站的SEO优化，可以促进关键词排名
    • 建议在网页中最多只有一个h1元素
    • 乱用不会带来好的权重，反而有可能被搜索引擎认为作弊，最后导致K站
    • logo优化：
      logo里面首先放一个h1标签，目的为了提权，告诉引擎这个地方很重要
h1里面在放一个连接，可以返回首页的，block给链接一个logo的背景图片
链接里面放文字，为了搜索引擎收录我们，但是文字不要显示出来。
      text-indent:-9999px，然后overflow：hidden 淘宝的做法
          直接给font-size : 0，就看不到文字了，     京东的做法
最后给链接一个title，这样鼠标放到logo上，就可以看到提示文字了
  • 网站优化三大标签:
    • 一.网页title标题
          title具有不可替代性,使我们内页的第一个重要标签,是搜索引擎了解网页的入口,
和对网页主题归属的最佳判断点.
  标题长度:
      Google,35个中文   Baidu,28个中文
  关键字分布:
      最先出现的词语权重越高
  关键字词频:
      主关键词出现3次,辅关键词出现一次.
  建议:
      首页标题:网站名(产品名)-网站的介绍
    • 二.Description:网站说明
          字符数含空格在内不超过120个汉字.
      补充在title和keywords 中未能充分表述的说明.
      用英文逗号 关键词1,关键词2
    • 三.keywords：关键字
          通常限制到6~8个

四、标签语义化

  • 什么是标签语义化？
    • 选择标签的时候要尽量让每一个标签都有正确的语义
  • 标签语义化的好处
    • 方便维护代码
    • 减少让开发者之间的沟通成本
    • 能让语音合成工具正确识别网页元素的用途，以便做出正确的反应
    • 让搜索引擎能够正确识别重要的信息

text/pain：普通文本传输

    • 传统的 表单提交

      • 弊端
        • 会进行页面的跳转（意味着服务器必须提前将一个页面写好，并且将写好的页面返回给前端，前端直接展示这个页面）

    • 前后端分离

        将数据传递给服务器，验证成功后，服务器会返回结果，需要前端解析这个数据，并且决定显示什么内容（前端渲染和前端路由 ）

分类

块级、行内级元素

  • 根据元素的显示（能不能在同一行显示）类型
    • 块级元素
      • 独占父元素一行
    • 行内级元素
      • 多个行内级元素可以在父元素的同一行中显示
      • 行内非替换元素
        • 不起作用：width、height、margin-top、margin-bottom
        • 效果特殊：padding-top、padding-bottom、上下方向的border
          • 上下会多出区域，但是不占据空间
      • 基线对齐
      • 行内级元素（包括inline-block元素）的代码之间如果有空格，会被解析为空格
        • 解决方法
          • 1.元素代码之间不要留空格
          • 2.注释掉空格
          • 3.设置父元素的font-size为0，然后在元素中重新设置自己需要的font-size

替换、非替换元素

  • 根据元素的内容（是否浏览器会替换掉元素）类型
    • 替换元素
      • 元素本身没有实际内容，浏览器根据元素的类型和属性，来决定元素的具体显示内容
      • 例img、input、iframe、video、embed、canvas、audio、object等
    • 非替换元素
      • 和替换元素相反，元素本身是有实际内容的，浏览器会直接将内容显示出来，而不需要根据元素类型和属性来判断到底显示什么内容

一、常见CSS属性

  • color: 前景色(不仅仅是文字，边框，贯穿线，下划线)
  • width、height: 不适用于非替换行内元素
  • 颜色
    • rgba (red,green,blue, alpha)
    • 注意 ：背景半透明是指盒子背景半透明，盒子里面的内容不受影响。

文本

  • direction：设置文本、表列水平溢出的方向
  • letter-spacing：每个字母的间距 => H e l l o W o r l d
    • 默认为0，负数会挤在一起
  • word-spacing：每个词的间距 => Hello World
    • 默认为0，负数会挤在一起
  • text-indent： 用于设置第一行内容的缩进
    • 单位最好为em，相对自身文字大小，例 text-indent: 2em
  • text-transform: 用于设置文字的大小写转换
    • capitalize: 将每个单词的首字符变为大写
    • uppercase: 将每个单词的所有字符变为大写
    • lowercase: 将每个单词的所有字符变为小写
    • none: 没有任何影响
  • text-decoration: 用于设置文字的装饰线
    • none
    • underline:下划线
    • overline:上划线
    • line-through:中划线（删除线）
  • text-align: 定义行内内容如何相对它的块父元素对齐
    • 注：text-align并不控制块元素自己的对齐，只控制它的行内内容的对齐
    • justify:文字向两侧对齐，对最后一行无效。
  • text-align-last:
    • justify:使文本最后一行也向两侧对齐
  • white-space
    • nowarp 防止文本换行
  • word-break
    • break-all：可以将单词断开

字体

  • font-size: 用来设置字体大小
    • 单位:px,em, 百分比，后面两个都是相对于父元素的字体大小而言
  • font-style：用于设置文字的常规、斜体展示
    • 斜体
    • 斜体
    • italic: 用字体的斜体显示(前提，font-family这种字体本身支持斜体)
    • oblique: 文本倾斜显示（让文字倾斜）
    • normal: 正常
  • font-variant：可以影响小写字母的显示形式
    • normal: 常规显示
    • small-caps: 将小写字母替换为缩小过的大写字母
  • font-family: 本质上是去本地操作系统上去寻找字体文件
  • line-height: 如果块级元素没有设置高度，只含文本，那么height是由行高撑起来的
    • 为什么文本需要行高？
      • 人的阅读习惯，行高决定阅读方式
    • 行高的严格定义：两行文字基线(baseline)之间的间距
  • font: 缩写属性
    • 顺序：font-style font-weight font-size font-family;
    • 中间用空格隔开，顺序不得改变，不需要的属性可以省略，但必须保留size和family。

背景

  • background
    • 一系列背景相关属性的缩写\
    • 常用格式
      • image position/size repeat attachment color
      • size可以省略，如果不省略，必须紧跟在position后
      • 其他随意
  • background-image
    • 背景图片会盖在background-color上（层叠）
    • 如果设置了多张背景图片，也是层叠
    • 注：如果设置了背景图片，元素没有具体的宽高，背景图片是不会显示出来的
  • background-size
    • auto:以背景图本身大小显示
    • cover：缩放背景图，以完全覆盖铺满元素
    • contain：缩放背景图，宽度或者高度福满元素，但是图片保持宽高比
    • 百分比、或者具体大小
  • background-repeat
    • no-repeat：不平铺
    • repeat-x：x轴平铺
    • repeat-y：y轴平铺
    • repeat：x和y都平铺
  • background-attachment
    • scroll：随着div的滚动（浏览器）的元素而滚动，背景一起滚动
    • local：随着div内容的滚动，背景一起滚动，需要div有滚动条
    • fixed：背景固定，不会随着div的滚动而滚动，但会随着div的不可见而不可见
  • background-position
    • 具体数值
    • 水平方向
      • left、center、right
    • 垂直方法
      • top、center、bottom
    • 如果只设置了一个方向，另一个方向默认是center

盒子模型

  • 标准盒子模型
    • conntent-box: 设置宽度和高度时只指定内容的宽高
  • IE盒子模型（IE8以下）
    • border-box: 设置宽度和高度时指定内容加内边距加边框
margin

  • auto

    • left、right将剩余的空间均分
    • top、bottom都为0

  • 上下边距折叠（外边距塌陷）

  • 垂直嵌套外边距合并的解决方案:

    1.给父元素定义一个上边框
2.给父元素定义一个内边距padding-top
3.给父元素一个overflow :hidden

  • 上下margin传递

    • margin-top传递
      • 如果块级元素的顶部线和父元素的顶部线重叠，那么这个块级元素的margin-top值会传递给父元素，即相当于父元素设置margin-top
    • margin-bottom传递
      • 如果块级元素的底部线和父元素的底部线重叠，并且父元素的高度是auto, 那么这个块级元素的margin-bottom值会传递给父元素，即相当于父元素设置margin-bottom
    • 如何防止出现传递问题？
      • 给父元素设置padding-top\padding-bottom
      • 给父元素设置border
      • 触发BFC：设置overflow为auto/hidden

    建议

    • margin：一般用来处理兄弟元素之间的间距
    • padding：一般用来处理父子元素之间的间距
padding

  • padding: 0 0 0 0;

    • 顺时针，上右下左

  • padding: 0 0 0;

    • 顺时针，上右下, 没有左，左跟随右

  • padding: 0 0;

    • 顺时针，上右，没有下左，跟随上右

  • 如果盒子没有宽度，则padding不会撑开盒子。

  • 行内元素只设置左右 内 外边距

  • border

    • border-style
      • none
      • dotted：原点
      • dashed：虚线（常用）
      • solid：实线（常用）
      • double：双实线
      • groove：先凹陷再凸起
      • ridge：先凸起再凹陷
      • inset：凸起
      • outset：凹陷

  • width

    • max-width
    • min-width

  • height

    • max-height
    • min-height

列表

  • list-style

表格

  • border-collapse
    • 合并边框

显示

  • display
    • block
      • 块级元素
    • inline
      • 行内级元素
    • none
      • 隐藏
      • 不保留空间
    • inline-block
      • 行内块元素
      • 跟block元素一样，可以嵌套任何元素
    • display的以下取值，等同于某些HTML元素
      • table：<table>，一个block-level表格
      • inline-table：<table>，一个inline-level表格
      • table-row：<tr>
      • table-row-group: <tbody>
      • table-header-group: <thead>
      • table-foot-group: <tfoot>
      • table-cell: <td>、<th>，一个单元格
      • table-caption: <caption> 表格的标题
      • list-item：li
  • visibility
    • visible：显示元素
    • hidden：隐藏元素
      • 虽然元素看不见，但是依然会保留空间
  • overflow
    • visible：超出的内容仍然显示
    • hidden：超出的内容隐藏
    • scroll：超出部分会隐藏，但是可以通过滚动条显示（滚动条区域也会占据空间，属于width、height）
    • auto：有超出部分，可以滚动，没有就正常显示
    • overflow-x、overflow-y：可以分别设置水平垂直方向
  • opacity
  • filter

定位

  • vertical-align
  • position
    • static
    • relative
    • absolute
      • 定位参照对象的宽度 = left + right + margin-left + margin-right + 绝对定位元素的实际占用宽度
      • 高度同上
    • fixed
  • left top right bottom
  • float
    • 浮动的规则
      • 浮动后，脱离文档流，朝着左（右）方向移动，直到自己的边界紧贴着包含块，或者其他浮动元素的边界上
      • 浮动元素不能与行内级元素层叠，行内级元素将会被浮动元素推出
      • 行内级元素、inline-block元素浮动后，其顶部将与所在行的顶部对齐
      • 浮动元素的边界不能超出包含块的边界
      • 浮动元素不能重叠
      • 浮动元素的顶端不能超过包含块的顶端，也不能超过之前所有浮动元素的顶端
  • clear

标准元素 => 浮动元素 => 定位元素

其他属性

outline

  • 表示元素的外轮廓
    • 不占据空间，默认显示在border的外面
    • 相关属性
      • outline-width、outline-style、outline-color

box-shadow

  • = inset? && {2, 4} && ?
  • 第一个：水平方向的偏移，正数往右偏移
  • 第二个：垂直方向的偏移，正数往下偏移
  • 第三个：模糊半径（blur radius）
  • 第四个：延伸距离
  • ：阴影的颜色，如果没有设置，就跟随color属性的颜色
  • inset：外框阴影变成内框阴影

text-shadow

  • 用法类似于box-shadow，用于给文字添加阴影属性

cursor 光标

  • auto：浏览器根据上下文决定指针的显示样式，比如根据文本和非文本切换指针样式
  • default：由操作系统决定，一般就是一个小箭头
  • pointer：一直小手，鼠标指针挪动到链接上默认就是这个样式
  • text：一条竖线
  • not-allowed：禁止

二、CSS选择器(selector)

  • 什么是CSS选择器
    • 按照一定的规则选出符合条件的元素，为之添加CSS样式
  • 分类

通用选择器

  • *{}

元素选择器

  • div {} p {}

类选择器

  • .nav {}

id选择器

  • #name {}

属性选择器

  • [title] {}
  • [name=”age”] {}
  • [title*=”one”] {}
    • 包含one的
  • [title^=”one”] {}
    • one开头的
  • [title$=”one”] {}
    • one结束的

组合

  • 后代选择器
    • div span {}
      • 包括直接、间接子元素
  • 子选择器
    • div>span {}
      • 只包括直接子元素
  • 相邻兄弟选择器
    • div+p {}
      • div后面紧挨着的p元素
  • 全体兄弟选择器
    • div~p {}
      • div后面的p元素
  • 交集选择器
    • div.one {}
      • 同时符合两个条件的元素
  • 并集选择器
    • div, .one {}
      • 满足任意一个

伪类

  • 动态伪类
    • 常用的 a
      • a:link 未访问的链接
      • a:visited 访问过的链接
      • a:hover 鼠标停留在上面
        • 注意，hover必须放在link跟visited后面才会生效
      • a:active 点击时的状态（鼠标点击未松开）
        • :active必须放在hover后面才能完全生效
      • 如果不想a有focus状态
        • tabindex = “-1”
          • 表示这个元素不能被键盘tab选中
        • a:focus { outline: none}
          • 去掉轮廓线，假装不能被选中
      • 细节： 如果直接给a元素设置样式，相当于给a元素的所有动态伪类都设置了
    • :focus 获取焦点的时候的状态
      • 因为a标签也有focus状态，所以建议编写顺序在hover之后，active之前
  • 目标伪类
    • :target {}
      • 选择一个ID与当前URL片段匹配的元素
  • 语言伪类
  • 元素状态伪类
    • 例如button的 enable 和 disabled
      • :disabled { color: red; }
  • 结构伪类
    • :nth-child()
      • n: 自然数
      • 2n => even ：偶数
      • 2n + 1 => odd : 奇数
      • -n + x ：前x位被选中
      • :first-child => :nth-child(1)
    • :nth-last-child() 倒着数
      • :last-child => :nth-last-child(1)
    • :nth-of-type()
      • 只计算同种类型的元素
      • :first-of-type => :nth-of-type(1)
    • :nth-last-of-type()
      • :last-of-type => :nth-last-of-type(1)
    • :only-child 是父元素的唯一的子元素
    • :only-of-type 是父元素的唯一的这种子元素
    • :root 根元素，即选中html标签
    • :empty 元素内容为空
  • 否定伪类
    • :not(x)
      • x是一个简单选择器
        • 元素，通用，属性，类，id~, 伪类(除否定伪类)
      • 除了div之外，所有的元素

伪元素

  • 为了区分伪元素和伪类，建议伪元素使用两个冒号

  • 默认伪元素为行内元素

  • 常用的伪元素

    • ::first-line
      • 选中第一行字母
      • 只有下列属性可以应用在::first-line伪元素
        • 字体属性，颜色属性，背景属性
        • word-spacing, letter-spacing, text-decoration, text-transform, line-height
    • ::first-letter
      • 选中第一个字母
      • padding margin height
    • ::before
      • 用来在一个元素的内容之前插入其他内容
      • content 里面存放内容，不能省略
        • 可以用来存放文字或者图片
    • ::after
      • 用来在一个元素的内容之后插入其他内容

html

  • 内容

      • 1、2、3、4、5

四、CSS的特性

继承

  • 常见的属性

    • color、font-size

  • 元素自己设置的元素，是优先级最高的

  • inherit

    • 强制继承父级的属性

  • 浏览器会自动识别哪些属性是继承过来的

  • 注意点

    • CSS属性继承的是计算值，并不是当初编写属性时的指定值（字面值）

      • .box1 {
  width: 50%;   // 250px
}
.box1 .box2 {
  width: inherit;   //250px
}

层叠

  • 基本层叠

    • 后面的把前面的覆盖掉
    • 前提
      • 使用相同的选择器

  • 权重

    • 继承或者* 0，0，0，0
    • 标签选择器 0，0，0，1
    • 类，伪类 0，0，1，0
    • ID 0, 1, 0, 0
    • 行内样式style=”” 1，0，0，0
    • !important 无穷大

  • 权重的叠加 ： 权重可以叠加，但是无进位。继承的权重为0

五、布局

flex布局

  • 当一个元素添加了flex属性，那么它的子元素将自动转化为块级元素
  • flex-grow
    • 可拉伸
  • flex-shrink
    • 可压缩
  • flex-basis
    • 当前元素的宽度
  • flex默认值
    • flex-grow: 0, flex-shrink: 1, flex-basis: auto
  • flex: 1
    • flex-grow: 1, flex-shrink: 1, flex-basis: 0%
  • flex: auto
    • flex-grow: 1, flex-shrink: 1, flex-basis: auto

BFC

  • BFC: block format context
  • 结界

如何触发BFC：

  • 浮动
  • 设置一个元素的overflow为非visible

脱标元素

什么元素会脱离标准流？

  • position: fixed; position: absolute; float

脱标元素的特点

  • 可以随意设置宽高
  • 宽高默认由内容决定
  • 不再受标准流的约束
  • 不再给父元素汇报宽高数据
