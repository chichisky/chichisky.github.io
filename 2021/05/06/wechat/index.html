小程序学习笔记

小程序特点概述

  • 没有DOM
  • 组件化开发、具备特定功能效果的代码集合
  • 体积小，单个压缩包体积不能大于2M，否则无法上线
  • 小程序四个重要的文件
    • *.js
    • *.wxml => view结构 => html
    • *.wxss => view样式 => css
    • *.json => view数据 => json文件
  • 小程序适配方案：rpx (responsive pixel 响应式像素单位)
    • 小程序适配单位：rpx
    • 规定任何屏幕下宽度为750 rpx
    • 小程序会根据屏幕的宽度不同自动计算rpx的大小
    • Iphone6 下：1 rpx = 1 物理像素 = 0.5 px

配置

全局

  • project.config.json

    • 项目的配置文件

  • sitemap.map

    • 用于配置小程序及其页面是否允许被微信索引

  • app.js

    • 全局必须要有一个app.js文件
      • 在其中必须调用一次App()，且只能调用一次
        • 用于注册小程序，其指定小程序的生命周期回调等

  • app.json

    • 用来对微信小程序进行全局配置

  • app.wxss

    • 小程序会在每个wxml页面最外层套上一层page
      • 可以用page来修改全局的一些样式配置

### 自定义组件

- 将组件的HTML和css样式添加到组件中，然后哪个页面需要使用，在相应的json文件中注册

  - ~~~json
    {
      "usingComponents": {
        "NavHeader":"/components/NavHeader/NavHeader"
      }
    }

  • 组件与页面的不同

    • 注册的页面会自动添加到app.json中的pages中，但组件不会
    • 使用组件需要在相应的页面中注册

event对象

向event传参的两种方式

  • id = value

    • 获取方式
      • event.currtnyTarget.id
    • 注： 通过id向event传参的时候，如果传的是number类型会自动转化为string

  • data-key = value

    • 获取方式
      • event.currtnyTarget.dataset.key

  • 不同点

    • id是唯一传参，但是key可以有很多个
