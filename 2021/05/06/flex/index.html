mediaQuery

什么是媒体查询

  • 概念：为不同尺寸的屏幕设定不同的css样式

  • device-width、device-height

    • 设备的宽度、高度

  • width、height

    • 浏览器的宽度、高度
.box1 {
  width: 200px;
  height: 100px;
}
// - screen 屏幕的意思
// - 以下代码屏幕
// + 200-300px之间时,box1的背景颜色为pink
// + 300-500px之间时,box1的背景颜色为purple
@media screen and (min-device-width: 200px) and (max-device-width: 300px) {
  .box1{
    background-color: pink;
  }
}
@media screen and (min-device-width: 300px) and (max-device-width: 500px) {
  .box1{
    background-color: purple;
  }
}

媒体查询的引入方式

  • 第一种：在css 文件内引用
@media screen and (min-device-width: 200px) and (max-device-width: 300px) {
  .box1{
    background-color: pink;
  }
}
  • 第二种：在style标签属性中引用
<head>
  <style media="(min-device-width: 200px) and (max-device-width: 300px)">
    .box1{
    	background-color: pink;
  	}
  </style>
</head>
  • 第三种：在link标签中引用
<head>
  <link href="css/base.css" rel="stylesheet" media="(min-device-width: 200px) and (max-device-width: 300px)">
</head>

flex

弹性盒子

  • 弹性盒子中的子元素，如果不做额外的大小设置，可能会随着父元素自适应改变

父对象设置

  • flex-direction

    • 作用：子元素在父元素盒子中的排列方式
    • row：默认值。按从左到右的顺序显示
    • row-reverse：与row相同，但是以相反的顺序
    • column：灵活的项目将垂直显示，按从上到下的顺序
    • column-reverse：与column相同，但是以相反的顺序

  • flex-wrap

    • 作用：子元素在父元素盒子中是否换行
    • nowrap：默认值，不换行、不换列
    • wrap：换行、或换列
    • wrap-reverse：换行或换列，但以相反的顺序
      • 交叉轴相反

  • justify-content

    • 作用：用来在存在剩余空间时，设置为间距的格式
    • flex-start：默认值，从左到右，挨着行的开头
    • flex-end：从右到左，挨着行的结尾
    • center：剧中显示
    • space-between：平均分布在该行上，两边不留占据空间
    • space-around：平均分布在该行上，两边留有一半的间隔空间

  • align-items

    • 作用：设置每个flex元素在交叉轴上的对齐方式(把每一行当成一个整体)
    • flex-start：位于容器的开头
    • flex-end：位于容器的结尾
    • center：居中显示

  • align-content

    • 作用：设置每个flex元素在交叉轴上的对齐方式(把多行当成一个整体)
    • flex-start：位于容器的开头
    • flex-end：位于容器的结尾
    • center：剧中显示
    • space-between：平均分布，两边不留占据空间
    • space-around：平均分布，两边留有一半的间隔空间

子对象设置

  • flex-basis
    • 设置弹性盒伸缩基准值（设置后，宽度将不再生效）
  • flex-grow
    • 设置弹性盒子的扩展比率
    • 增加量：（父元素的宽度 - 所有子元素的宽度）/ 所有子元素的扩展比率之和 *单个子元素的扩展比率
  • flex-shrink
    • 设置弹性盒子的缩小比率
    • 默认有一个缩小比率，均为1，不会超出
    • 减少量：（所有子元素的宽度 - 父元素的宽度）/ 所有子元素的扩展比率之和 *单个子元素的扩展比率
  • flex
    • flex-grow、flex-shrink、flex-basis的缩写
    • flex默认值
      • flex-grow: 0, flex-shrink: 1, flex-basis: auto
    • flex: 1
      • flex-grow: 1, flex-shrink: 1, flex-basis: 0%
    • flex: auto
      • flex-grow: 1, flex-shrink: 1, flex-basis: auto
    • flex: none
      • flex-grow: 0, flex-shrink: 0, flex-basis: auto
    • flex: 0%
      • flex-grow: 1, flex-shrink: 1, flex-basis: 0%
    • flex: 100%
      • flex-grow: 1, flex-shrink: 1, flex-basis: 100%

案例三

rem的使用方法

  • 概念：指相对于根元素的字体大小的单位
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <script>
    var c = () => {
      let w = document.documentElement.clientWidth; /* 获取屏幕尺寸的大小*/
      let n = (20 * (w/360) > 40 ? (40 + 'px'): (20 * (w/360) + 'px'));
      document.documentElement.style.fontSize = n;
    }
    window.addEventListener('read', c);
    window.addEventListener('resize', c);
  </script>
  <style>
    div {
      font-size: 1rem;
    }
  </style>
</head>

<body>
  <div>
    123
  </div>
</body>

</html>

案例四

  • 自适应布局
    • 不同设备对应不同的HTML
    • 局部自适应
  • 一句话：不同的设备用不同的页面或局部伸缩
  • 设计思路：判断设备的类型或控制局部的变化
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>

</head>

<body>
</body>
<script>
  var redirect = () => {
    let userAgent = navigator.userAgent.toLowerCase();  /** 获取设备信息*/
    let device = /ipad|iphone|midp|rv:1.2.3.4|ucweb|android|windows ce|windows mobile/;
    if (device.test(userAgent)) {
      window.location.href = 'move.html'
    } else {
      window.location.href = 'pc.html'
    }
  }
  redirect();
</script>

</html>

- 布局特点：为了保证在各种屏幕上的不失真，就要根据实际屏幕宽度做等比例换算
