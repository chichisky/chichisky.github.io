TypeScript学习笔记

1. TypeScript 简介

  • TypeScript是JavaScript的超集

  • 相对于JavaScript来说，Typescript的优势

    • 类型化思维方式：使得开发更加严谨，提前发现错误，减少改bug时间
    • 类型系统提高了代码的可读性，并使维护和重构代码更容易
    • 补充了接口，枚举dd等大型应用时JS缺失的功能。

  • 类型注解：是一种为变量添加类型约束的方式

    ​ 可以用|来连接，同时定义多种变量类型

2. TypeScript 基本使用

变量类型

类型 例子 描述
number 1，-33 任意数字
string ‘x’, ‘o’ 任意字符串
boolean true, false 布尔值true或false
字面量 其本身 限制变量的值就是该自变量得值
any * 任意类型
unknown * 类型安全的any
void 空值(undefined) 没有值，或undefined
never 没有值 不能是任何值
object {} 任意对象
array [] 任意数组
tuple [4,5] 元素，TS新增类型，固定长度数组
enum enum(1,b) 枚举，TS中新增类型

  • unknown 和 any 的区别

    • 如果把any变量赋给其他变量，则会改变其他变量的变量类型为any
    • unknown无法改变其他变量的变量的类型

  • never与void的区别 ：void也算是一种返回类型，但是never不会返回

// never的使用场景
function (): never {
  throw new Error('报错了！')
}
  • tuple 固定长度的数组
let b: [string, string]
  • 类型的别名
// 可以用type来给类型起别名，适用于字面量类型
let i : 1|2|3|4|5
type myType = 1|2|3|4|5
let i : myType

2.1 TypeScript 与 VScode

//   安装typescript
npm i -g typescript

//   将TS文件转换成JS文件
tsc 要转换的文件

//   监视TS文件的变化并随之改变JS文件
tsc --watch 要监视的文件
//   简化typescript在node环境执行的步骤
npm i -g ts-node

//	 断点调试，在当前目录下载
npm i ts-node typescript
  • include
    • 用来指定什么路径下的文件被解析
  • compilerOptions
    • 编辑选项是配置文件中非常重要也比较复杂的配置选项
    • 在compilerOptions中包含多少子选项，用来完成对编译的设置
      • target
        • 设置ts代码编译的目标版本
      • module
        • 指定要使用的模块化的规范
      • lib
        • 指定项目中要使用的库
      • outDir
        • 指定解析后的js文件的所在目录
      • outFile
        • 将代码合并到一个文件
      • allowJs
        • 是否对JS文件进行编译，默认是false
      • checkJS
        • 是否检查JS代码是否符合语法规范，默认是false
      • removeComments
        • 是否移除注释
      • noEmit
        • 不生成编译后的文件
      • noEmitOnError
        • 当有错误时，不生成编译后的文件
      • strict
        • 严格模式的总开关，开启之后，所有严格模式都会开启
      • alwaysStrict
        • 用来设置编译后的文件是否使用严格模式
      • noImplicitAny
        • 不允许出现隐式any类型
      • noImplicitThis
        • 不允许不明确类型的this
      • strictNullChecks
        • 严格的检查空值

  • 配置webpack.config.js
// 引入path包
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin');
const { CleanWebpackPlugin } = require('clean-webpack-plugin');

//webpack所有的配置信息都应该卸载module.exports中
module.exports = {
  // 指定入口文件
  entry: "./src/index.ts",
  
  // 指定打包后的目录
  output: {
  	path: path(__dirname, 'dist'),
    filename: 'bundle.js',
    // 告诉webpack，不使用箭头函数
    environment: {
      arrowFunction: false
    }
	},
  
  // 指定webpack打包时要使用的模块
  module: {
    // 指定加载的规则
    rules: [
      {
        //test指定的是规则生效的文件
        test: /\.ts$/,
        // 要使用的loader
        use: [
          // 配置babel
          {
            // 指定加载器
            loader: "babel-loader",
            // 设置babel
            options: {
              // 指定预定义的环境
              presets: [
                [
                  // 指定环境的插件
                  "@babel/preset-env",
                  // 配置信息
                  {
                    // 要兼容的目标浏览器
                    targets: {
                      "chrome": "88"
                    },
                    // 指定corejs的版本
                    "corejs": "3",
                    // 使用corejs的方式 usage 表示按需加载
                    "useBuiltIns": "usage"
                  }
                ]
              ]
            }
          }, 
          'ts-loader'
        ],
        // 要排除的文件
        exclude: /nodule_modules/
      }
    ]
  },
  // 配置webpack插件
  plugins: [
    // 执行build时，先删除dist内文件，再重新生成，防止原来的文件存在
    new CleanWebpackPlugin(),
    new HtmlWebpackPlugin({
      title: "这是一个我自定义的title"
    }),
  ],
  // 用来设置引用模块
  resolve: {
    // 告诉webpack,以.ts和.js的文件是可以被当作模块使用的
    extensions: ['.ts', '.js']
  }
};

// 语法二
let names: string[] = new Array()  //不推荐

foods[2] = '包子'

foods[2] = '包子'

foods[foods.length] = '油泼面'  // 往数组的最后添加一个元素

2.4 函数

  • 注：如果没有指定函数的返回值，那么默认函数的返回值为void（空，什么都没有）
// 可以在函数声明时，指定函数返回值的类型
function fn(): 类型注解 {
  return 返回值
}

//参数的类型注解  如果一开始不清楚参数的类型，可以采用匿名函数获取参数类型
function fn(event: MouseEvent){
  event.target
}

2.5 对象

  • 对象：一组相关属性和方法的集合，并且是无序的
    • 在属性后面加?, 表示属性可选
// 对象属性的类型注解
let person: {
  name: string,
  age?: number
}

//  对象赋值
person = {
  name: 'zhangsan',
//  age: 18
}

// [propName: string]: any 表示任意类型的属性
let a = {[propName: string]: any}
a = {name: 'zs', age: 18}

// 对象方法的类型注解
// 分为是否有参数、返回值两种类型
let person: {
  sayHi: () => void,
  sing: (name: string) => void
  sun: (num1: number, num2: number) => number
}

2.7 类型断言

  • 由于querySelector()通过id选择器获取DOM元素时，拿到的元素类型都是Element
  • 导致问题：无法访问标签的其他属性，类似img的src属性， 类型断言可以解决
// 用法1 变量 as 类型
let img = document.querySelector('#img') as HTMLImageElement

// 用法2 <类型>变量
<string>img

2.8 枚举 enum

  • 使用方法
enum currentPlayer {x, o};
let userPlayer : currentPlayer = currentPlayer.x

  • 使用场景

    当变量的值，只能是固定的几个值之一，应该采用枚举实现

  • 总结

    枚举是组织有关联数据的一种方式

    枚举是一堆有名字的只读常量的集合

  • 注意：枚举中的成员只是可读的，因此，只能访问，不能赋值

2.9 类

// 1.声明普通类
class A {
  name: string;
  age: number;
  
  constructor (name: string, age: number) {
    this.name = name;
    this.age = age
  }
}
  • 直接定义的属性是实例实行，需要通过类的实例去访问
  • static定义的为静态属性，可以直接通过类型调用
  • readonly开头的属性表示一个可读的属性，不可以修改
  • abstract开头的为抽象类，不能创建实例
    • 抽象方法只能定义在抽象类和接口中，抽象方法没有方法体，子类必须重写抽象方法
// 抽象类的使用
abstract class Animal {
  name: string;
  age: number;
  abstract sayHello():void;
}

// 继承
class Dog extends Animal {
  name: "xixi";
  age: 2;
  sayHello() => {console.log('汪汪汪汪！')}
}
  • 接口
    • 接口可以重复声明，所有重复声明加在一起为接口实际内容
    • 接口内部都是抽象属性和抽象方法
interface IUser {
  name: string,
  age: number
}

// 实现接口
let p1: IUser = {
  name: 'zhangsan',
  age: 18
}

const p2 implements IUser {
  name: 'zhangsan',
  age: 18
}
  • 属性的封装
// public 在任何地方都可以访问
// private 只能在类内部进行访问
// protected 受包含的属性，只能在当前类或者子类中使用

// private

// 注意：在TS中，标识符不能相同，所以属性名，跟get，set方法名不能相同
class A {
  constructor(private _name: string, private _age: number){}

  get name() {
    return this._name;
  }
  set name(name: string) {
    this._name = name
  }
}
const a = new A('zs', 14);
console.log(a.name);

// 语法糖
class A {
  constructor(public name: string, public age: number)
